### "What is RAG and how does it work?"

"RAG stands for Retrieval Augmented Generation. The core idea is: instead of relying on what the LLM memorized during training, you retrieve relevant context at query time and inject it into the prompt.

The pipeline has three parts. First, indexing — you take your documents, chunk them, convert each chunk into an embedding using a model like text-embedding-3-small, and store those vectors in a vector database like pgvector or Pinecone. Second, retrieval — when the user asks a question, you convert that query into an embedding too, then do a cosine similarity search to find the most relevant chunks. Third, generation — you pass those retrieved chunks as context into the LLM prompt and let it generate a grounded answer.

I've implemented this in both Auren and Mnemo. In Mnemo specifically, I added a decay scoring layer on top of cosine similarity so more recent memories are weighted higher, and a contradiction detection step that flags if a new memory conflicts with an existing one."

---

### "What is a vector database and why do we use it?"

"A regular database stores structured data and lets you filter by exact values. A vector database stores high-dimensional numerical vectors — embeddings — and lets you search by semantic similarity. The key operation is approximate nearest neighbor search.

When you embed text, semantically similar text ends up close together in vector space. So 'how to cook pasta' and 'pasta recipe' will be near each other even though they share no exact words. That's something a SQL LIKE query cannot do.

I use pgvector — it's a PostgreSQL extension that adds vector types and operators. The index type I use is HNSW — Hierarchical Navigable Small World — which gives you sub-linear search time even on large datasets."

### "Explain the Transformer architecture"

"The Transformer came out of the 'Attention Is All You Need' paper from Google in 2017. The key insight was: you don't need RNNs to process sequences. Attention alone is enough.

The architecture has an encoder and decoder. The encoder takes input tokens, converts them to embeddings, adds positional encoding so the model knows token order, and then passes them through multi-head self-attention layers followed by feedforward layers. Self-attention lets every token attend to every other token — so it captures long-range dependencies that RNNs struggled with.

The decoder does the same but adds one extra step — cross-attention, which lets each output token attend to the encoder's representations.

BERT uses only the encoder, trained with masked language modeling. GPT uses only the decoder, trained to predict the next token. For chat models you mostly deal with GPT-style architectures."

### "What is the difference between fine-tuning, prompt engineering, and RAG?"

"These are three different levers to make an LLM do what you want.

Prompt engineering is the lightest — you craft your input carefully, maybe add examples in the prompt, structure it as chain-of-thought or few-shot. No training required. Best for general-purpose tasks where the model already has the knowledge.

RAG gives the model access to external knowledge at inference time. You don't change the model weights — you just inject relevant context into the prompt. Best when the model lacks specific domain knowledge or your data changes frequently.

Fine-tuning actually updates the model's weights using your dataset. The model learns your style, your format, your specific vocabulary. Best when you need consistent output format, domain-specific behavior, or you have lots of labeled examples. It's expensive and requires GPU access.

For most startup use cases, RAG is the right choice. Fine-tuning makes sense when you've already exhausted prompt engineering and RAG."

### "What is OOP? Explain with Python examples"

"OOP stands for Object Oriented Programming. It's a paradigm where you model real-world entities as objects that have attributes (data) and methods (behavior).

Four pillars.

Encapsulation — wrapping data and the methods that operate on it inside a class, and controlling access. In Python, you prefix with underscore for convention-based privacy. Not enforced at language level like Java but respected by convention.

Inheritance — a class can inherit from another class and reuse its attributes and methods. Python supports multiple inheritance. MRO (Method Resolution Order) using C3 linearization determines which parent's method gets called.

Polymorphism — same method name, different behavior. In Python this is mostly duck typing — if it has the method, it works. You don't need explicit interfaces.

Abstraction — hiding implementation details and exposing only what's necessary. Python's ABC module lets you define abstract classes with abstract methods that subclasses must implement."

---

### "What are ACID properties in DBMS?"

"ACID is a set of properties that guarantee database transactions are reliable.

Atomicity — a transaction is all or nothing. If you're transferring money between accounts, either both the debit and credit happen, or neither does.

Consistency — the database must go from one valid state to another. No rules are violated after a transaction commits.

Isolation — concurrent transactions don't interfere with each other. Each transaction sees a consistent snapshot.

Durability — once a transaction commits, it stays committed even if the system crashes. It's written to disk, not just in memory.

SQL databases are ACID-compliant by default. NoSQL databases often trade some of these (especially isolation and consistency) for scalability and speed."

---

### "Explain normalisation"

"Normalization is the process of organizing a database to reduce redundancy and avoid anomalies — insert, update, and delete anomalies.

First Normal Form (1NF): every column has atomic values. No arrays or repeating groups inside a cell.

Second Normal Form (2NF): 1NF plus every non-key column must depend on the entire primary key, not just part of it. This matters for composite primary keys.

Third Normal Form (3NF): 2NF plus no transitive dependency. Non-key columns should not depend on other non-key columns.

BCNF (Boyce-Codd Normal Form): stricter version of 3NF. Every determinant must be a candidate key.

In practice, most production databases normalize to 3NF and sometimes denormalize deliberately for read performance."

### "What is deadlock? When does it occur?"

"A deadlock is a state where two or more processes are stuck, each waiting for a resource held by the other, so none of them can proceed.

Four conditions must all be true simultaneously for a deadlock to occur. Mutual exclusion — at least one resource is non-shareable. Hold and wait — a process holding one resource is waiting for another. No preemption — resources can't be forcibly taken from a process. Circular wait — there's a cycle of processes each waiting on the next.

To prevent deadlock you break one of these conditions. Common strategies: impose a fixed resource ordering to break circular wait, use resource allocation graphs, or use the Banker's Algorithm which checks before granting a resource whether the system will remain in a safe state."

---

### "What is the difference between process and thread?"

"A process is an independent program in execution. It has its own memory space — stack, heap, code, data. Processes are isolated from each other. If one crashes, others are unaffected.

A thread is the smallest unit of execution within a process. Threads in the same process share memory — they share the heap and the code segment, but each has its own stack and registers. That shared memory is what makes threads faster to communicate but also harder to program safely — you need synchronization to avoid race conditions.

Context switching between threads is cheaper than between processes because threads share the same memory space."

### "Write a query: find the second highest salary from an Employee table"

"There are two clean ways.

First approach using subquery:
SELECT MAX(salary) FROM Employee WHERE salary < (SELECT MAX(salary) FROM Employee);

Second approach using DENSE_RANK or LIMIT:
SELECT salary FROM Employee ORDER BY salary DESC LIMIT 1 OFFSET 1;

The subquery approach works across all SQL dialects. The LIMIT/OFFSET approach works in MySQL and PostgreSQL. If duplicates matter, use DENSE_RANK in a window function."
