🏢 Company Preparation Cockpit

Master hiring flows, critical rounds, and high-yield system design focus areas for top-tier companies.

6 Major Firms 24+ Rounds
G
Google
Tier 1
01 Technical Phone Screen: Deep focus on time/space complexity analysis and modular code.
02 Coding Rounds (x3): Hard algorithmic puzzles (graphs, dynamic programming, sorting).
03 System Design: MapReduce, BigTable architectures, or rate limiting at massive scale.
⚠️ INTERVIEWER TRAP: Jumping straight to coding without discussing edge cases and alternative approaches. Ask clarifying questions first!
Dijkstra & A* Tries Distributed Systems
M
Microsoft
Tier 1
01 Online Assessment (OA): Codility assessment containing 3 coding challenges.
02 DS & Algos (x2): Tree traversal, design patterns, strings, and OOP design.
03 System Design & Architecture: Cloud migration patterns, load balancer setups, database sharding.
⚠️ COMMON MISTAKE: Weak OOP structures. Microsoft values clean class structures, encapsulation, and interfaces.
OOPs Patterns Binary Trees DBMS & SQL
A
Amazon
Tier 1
01 Online Assessment: Coding task + Work Simulation test evaluating LP metrics.
02 Coding Rounds (x2): Focus on lists, stacks/queues, heaps, and tree maps.
03 System Design & LP (x2): Pub-Sub event architectures, DynamoDB scaling, and LP scenarios.
⚠️ CRITICAL NOTE: 50% of the evaluation rests on Amazon's 16 Leadership Principles. Always back answers with the STAR method.
Heaps & Priority Queues Leadership Principles NoSQL Scaling
U
Uber
Startup Tier 1
01 OA / Phone Screen: Advanced algorithm screen with speed-optimized benchmarks.
02 Low Level Design (LLD): Implementation of thread-safe systems (e.g. Rate Limiter, Ride Engine).
03 High Level Design (HLD): Geo-spatial indexing, dynamic pricing engine, real-time message brokers.
⚠️ INTERVIEWER TRAP: Ignoring concurrent write race conditions. Always address multi-threading locks and eventual consistency.
H3 Geo-Spatial Indexing Concurrency Locks Redis & WebSockets
S
Swiggy
SaaS / Unicorn
01 HackerEarth Test: 2 core algorithmic questions + SQL queries.
02 Machine Coding (LLD): Complete working CLI app in 90-120 minutes (e.g. food delivery system).
03 High Level Design: Microservices orchestration, real-time order tracking, cache synchronization.
⚠️ MACHINE CODING RULE: Focus on clean architecture (SOLID) and extensibility rather than just a quick and dirty solution.
Microservices Machine Coding LLD Message Queues
R
Razorpay
Fintech Leader
01 DS/Algo Screen: Core focus on linked lists, queues, caching implementations.
02 System Design & LLD: Payment gateway simulation, handling idempotency, and audit trails.
03 Culture Fit Round: Collaborative engineering mindset, handling production incidents.
⚠️ CRITICAL THEORY: Understand **idempotency** (API-key deduction tracking) and **distributed database transactions** thoroughly.
Idempotent APIs 2-Phase Commits SQL ACID Rules
