1. Network Layer Functions and Protocols

Functions

  • Addressing:
    • Assigns logical addresses, such as IP addresses, to devices in a network.
    • These addresses uniquely identify devices, enabling communication across different networks.
    • Logical addresses differ from physical addresses (e.g., MAC addresses) and are essential for routing packets.
  • Routing:
    • Determines the most efficient path for data packets to travel from the source to the destination.
    • Routing algorithms, such as Dijkstra's and Bellman-Ford, calculate optimal routes based on metrics like hop count, latency, or bandwidth.
    • Routers use these algorithms to direct packets across networks efficiently.
  • Packet Forwarding:
    • Transfers data packets from one network to another based on routing table entries.
    • Ensures packets reach the correct destination by forwarding them through intermediary devices like routers.
  • Fragmentation and Reassembly:
    • Splits large data packets into smaller fragments to fit the MTU (Maximum Transmission Unit) of the network.
    • Reassembles fragments at the destination to reconstruct the original packet.
    • This process ensures data can traverse networks with different MTU sizes.

Protocols

  • IP (Internet Protocol): Handles addressing and routing of packets.
    • IPv4: Utilizes 32-bit addresses, offering approximately 4.3 billion unique addresses.
    • IPv6: Uses 128-bit addresses, significantly increasing address capacity and incorporating security features.
  • ICMP (Internet Control Message Protocol): Used for network diagnostics and error reporting, such as determining if a host is reachable via ping.
  • ARP (Address Resolution Protocol): Maps IP addresses to MAC (Media Access Control) addresses for local network communication.
  • BGP (Border Gateway Protocol): Manages routing between autonomous systems, essential for internet-scale routing.

2. Transport Layer Functions and Protocols

Functions

  • Segmentation and Reassembly:
    - Divides large data into smaller segments for efficient transmission.
    - Reassembles segments at the destination to reconstruct the original data.
  • Error Detection and Recovery:
    - Identifies corrupted or lost segments during transmission.
    - Implements retransmission mechanisms to recover lost data and ensure integrity.
  • Flow Control:
    - Manages data flow to prevent congestion and buffer overflow at the receiver.
    - Ensures optimal performance by adjusting the rate of transmission.
  • Port Addressing:
    - Uses port numbers to identify specific applications or services.
    - Enables simultaneous communication with multiple services, such as web servers (port 80) and email servers (port 25).

Protocols

  • TCP (Transmission Control Protocol): Reliable, connection-oriented protocol ensuring ordered and error-free delivery of data.
  • UDP (User Datagram Protocol): Lightweight, connectionless protocol offering fast but unreliable data transmission.
  • SCTP (Stream Control Transmission Protocol): Combines features of TCP and UDP, supporting multi-streaming and enhanced reliability.

3. Session Layer Protocols

Functions

  • Establishes, maintains, and terminates communication sessions between devices or applications.
  • Synchronizes data exchange by introducing checkpoints to recover from disruptions.
  • Supports full-duplex (simultaneous two-way communication) and half-duplex (one-way at a time) operations.

Protocols

  • NetBIOS (Network Basic Input/Output System): Provides session-layer services for communication between computers.
  • PPTP (Point-to-Point Tunneling Protocol): Used for establishing secure VPN sessions by encapsulating data packets.
  • SOCKS (Socket Secure): Handles proxy server sessions to enable secure and anonymized communication.

4. Connectionless vs. Connection-Oriented Protocols

Connection-Oriented

  • Establishes a dedicated connection between sender and receiver before data transfer begins.
  • Example: TCP, which ensures reliability through acknowledgments and retransmissions.

Connectionless

  • Sends data packets directly without establishing a connection beforehand.
  • Example: UDP, which is faster but does not guarantee delivery or order of packets.

Comparison

Feature Connection-Oriented Connectionless
Reliability Reliable (acknowledgments and retransmissions) Unreliable (no guarantees of delivery)
Setup Requires connection setup (e.g., handshake) No setup required
Overhead High (control messages and session maintenance) Low (minimal control overhead)
Use Case File transfer, email, web browsing Streaming, online gaming, DNS queries

Applications

  • Network Layer: Used in routing packets across LANs, WANs, and the internet to ensure efficient data delivery.
  • Transport Layer: Critical for ensuring reliable or fast delivery of data, such as in video streaming, file transfers, or real-time communication.
  • Session Layer: Essential for maintaining persistent sessions, such as in online gaming or remote file sharing applications.
  • Connectionless Protocols: Ideal for real-time applications like VoIP or live video streaming, where speed is prioritized over reliability.
  • Connection-Oriented Protocols: Best suited for applications requiring data integrity and reliability, such as secure file transfers and email communication.
