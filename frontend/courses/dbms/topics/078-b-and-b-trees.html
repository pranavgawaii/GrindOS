2. Characteristics of B-Trees

  • Balanced Structure: All leaf nodes are at the same level, ensuring balanced height.
  • Multiple Keys per Node: Each node can store more than one key.
  • Sorted Order: Keys in each node are maintained in sorted order.
  • Dynamic Size: Adjusts its structure automatically with inserts and deletes.
  • Efficient Disk Access: Optimized for systems with high disk access latency.

3. Structure of B-Trees

  • Internal Nodes: Store keys for partitioning data and pointers to child nodes.
  • Leaf Nodes: Store actual data or pointers to the data records.

4. Operations in B-Trees

  • Search: Traverse from the root to the leaf nodes, comparing keys at each level.
  • Insert: Insert new keys into the appropriate node, splitting the node if it exceeds its maximum capacity.
  • Delete: Remove keys while maintaining balance, either by borrowing keys from sibling nodes or merging nodes.

Example: B-Tree Construction

Keys to insert: [10, 20, 5, 6, 12, 30, 7, 17]

Step-by-Step Process:
1. Insert 10 and 20:
   [10, 20]

2. Insert 5:
   [5, 10, 20]

3. Insert 6 (causes split):
        [10]
       /   \
    [5, 6] [20]

4. Insert 12:
        [10]
       /   \
    [5, 6] [12, 20]

5. Insert 30 (causes split):
         [10, 20]
       /    |    \
    [5, 6] [12] [30]

6. Insert 7:
         [10, 20]
       /    |    \
 [5, 6, 7] [12] [30]

7. Insert 17:
         [10, 20]
       /    |    \
 [5, 6, 7] [12, 17] [30]

5. Structure of B+ Trees

B+ Trees are an extension of B-Trees where:

  • Internal Nodes: Store only keys and pointers to child nodes.
  • Leaf Nodes: Store all actual data and are linked sequentially for fast range queries.

6. Operations in B+ Trees

  • Search: Similar to B-Trees but requires traversal to the leaf node to access actual data.
  • Insert: Insert keys into leaf nodes, splitting if necessary, and updating parent nodes.
  • Delete: Remove keys from leaf nodes, borrowing or merging as needed while maintaining balance.

Step-by-Step B+ Tree Construction

Keys to insert: [10, 20, 5, 6, 12, 30, 7, 17]

1. Insert 10 and 20:
   [10, 20]

2. Insert 5:
   [5, 10, 20]

3. Insert 6 (causes split):
        [10]
       /   \
    [5, 6] [20]

4. Insert 12:
        [10]
       /   \
    [5, 6] [12, 20]

5. Insert 30 (causes split):
         [10, 20]
       /    |    \
    [5, 6] [12] [30]

6. Insert 7 (linked leaf nodes):
         [10, 20]
       /    |    \
 [5, 6, 7] [12] [30]

7. B+ Trees vs. B-Trees

Feature B-Tree B+ Tree
Internal Nodes Store keys and pointers to data Store only keys and child pointers
Leaf Nodes Store keys and/or data Store all data and are linked
Range Queries Less efficient Highly efficient
Access Speed Slightly slower for large datasets Faster for sequential data access

8. Advantages of B-Trees and B+ Trees

Advantages of B-Trees:

  • Efficient for point lookups.
  • Balanced structure ensures consistent performance.
  • Reduces disk I/O by grouping multiple keys in each node.

Advantages of B+ Trees:

  • Optimized for range queries.
  • Linked leaf nodes allow sequential data access.
  • Better suited for disk-based storage systems.

Conclusion

Both B-Trees and B+ Trees are essential data structures for database systems. While B-Trees provide efficient point lookups, B+ Trees excel in handling range queries and sequential data access. Understanding their characteristics, structure, and operations is crucial for database optimization.
