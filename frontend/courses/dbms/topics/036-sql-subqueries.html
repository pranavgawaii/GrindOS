Subqueries

A Subquery is a query nested inside another SQL query. It is often used to retrieve data that will be utilized in the main (outer) query as a condition for filtering, grouping, or joining data. Subqueries can return single values, lists, or entire result sets, depending on their usage.

Key Points of Subqueries

  • Nested Queries: Subqueries are embedded within other SQL statements.
  • Flexible Placement: They can be used in SELECT, WHERE, FROM, and HAVING clauses.
  • Scalable Results: Subqueries can return a single value, a list, or a complete table.


Subqueries in SELECT Statements

Subqueries in SELECT statements allow retrieval of related data from another table and its usage as a computed column in the main query.

Example

Consider the following tables:

Product Table
Product_ID Product_Name Price
1 Laptop 1200
2 Smartphone 800
3 Mouse 20
Sales Table
Sale_ID Product_ID Quantity
1 1 5
2 2 10
3 1 2 
SELECT Product_Name,
       (SELECT SUM(Quantity) 
        FROM Sales 
        WHERE Sales.Product_ID = Product.Product_ID) * Price AS Total_Revenue
FROM Product;

Result:

Product_Name Total_Revenue
Laptop 8400
Smartphone 8000
Mouse NULL

Subqueries in WHERE Clause

Subqueries in the WHERE clause filter data based on conditions met by related data in another table.

Example

SELECT Product_Name
FROM Product
WHERE Product_ID IN (SELECT Product_ID FROM Sales);

Result:

Product_Name
Laptop
Smartphone

Subqueries in FROM Clauses

Subqueries in the FROM clause treat the result of a subquery as a temporary or derived table.

Example

SELECT Product_Name, Total_Sales
FROM (SELECT Product.Product_Name, 
             SUM(Sales.Quantity * Product.Price) AS Total_Sales
      FROM Product
      JOIN Sales ON Product.Product_ID = Sales.Product_ID
      GROUP BY Product.Product_Name) AS SalesSummary
WHERE Total_Sales > 5000;

Result:

Product_Name Total_Sales
Laptop 8400
Smartphone 8000

Subqueries with EXISTS and NOT EXISTS

EXISTS and NOT EXISTS check for the presence or absence of rows that meet certain conditions. EXISTS returns TRUE if the subquery returns any rows, and NOT EXISTS returns TRUE if the subquery returns no rows.

Example

SELECT Product_Name
FROM Product
WHERE NOT EXISTS (SELECT * 
                  FROM Sales 
                  WHERE Sales.Product_ID = Product.Product_ID);

Result:

Product_Name
Mouse

Subqueries with IN and NOT IN Operators

Subqueries with IN and NOT IN compare a column to a list of values returned by a subquery.

Example

SELECT Product_Name
FROM Product
WHERE Product_ID IN (SELECT Product_ID 
                     FROM Sales 
                     WHERE Quantity >= 10);

Result:

Product_Name
Smartphone

Summary Table of SQL Subquery Usages

Subquery Type Description Example Use Case
SELECT Subquery Use subquery result as a computed column Calculating total revenue per product
WHERE Subquery Filter data based on related table data List products with sales
FROM Subquery Use subquery as a derived table Select products with total sales above a threshold
EXISTS / NOT EXISTS Check for the presence/absence of records List products with or without sales records
IN / NOT IN Filter based on matching values in a list List products with specific sales quantities
