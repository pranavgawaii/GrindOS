What are Stored Procedures?

A Stored Procedure is a precompiled collection of SQL statements and optional control-of-flow statements stored under a name and processed as a unit. Stored procedures are created and stored in the database, allowing users to execute a sequence of SQL commands with a single call.

Real-Life Analogy:

Imagine a recipe for baking a cake. Each time you want to make the cake, instead of remembering each step and measurement, you simply follow the recipe. Similarly, a stored procedure in a database is like a recipe that the database can execute repeatedly without needing to rewrite the SQL statements.

Creating and Using Stored Procedures

Stored procedures can be created using SQL CREATE PROCEDURE or CREATE FUNCTION statements (depending on the database system). They can be invoked with a simple CALL statement, which executes all the SQL commands within the procedure.

Example: Creating a Simple Stored Procedure

Consider a Student Database where we want to get a student’s information based on their ID.

	CREATE PROCEDURE GetStudentInfo (IN student_id INT)
	BEGIN
	SELECT * FROM students WHERE id = student_id;
	END;

Using the Stored Procedure:

	CALL GetStudentInfo(101);

Output (Assuming student with ID 101 exists):

id name age grade
101 John Doe 20 A

This stored procedure, GetStudentInfo, takes a student ID as input and retrieves that student’s information from the students table.

Parameters in Stored Procedures

Stored procedures can accept parameters to make them flexible and reusable. There are three types of parameters:

  • IN: The procedure takes input values.
  • OUT: The procedure returns values to the caller.
  • INOUT: The procedure takes input and can also modify and return the value.

Example: Stored Procedure with Parameters

Consider a Banking System where we want to update a customer’s balance and return the updated balance.

	CREATE PROCEDURE UpdateBalance (IN cust_id INT, IN deposit_amount DECIMAL(10,2), OUT new_balance DECIMAL(10,2))
	BEGIN
	UPDATE accounts SET balance = balance + deposit_amount WHERE customer_id = cust_id;
	SELECT balance INTO new_balance FROM accounts WHERE customer_id = cust_id;
	END;

Using the Stored Procedure:

	CALL UpdateBalance(101, 500.00, @updated_balance);
	SELECT @updated_balance;

In this example:

  • cust_id and deposit_amount are IN parameters (inputs).
  • new_balance is an OUT parameter, which returns the updated balance.

Benefits of Stored Procedures

Stored procedures provide several advantages in database management:

  • Reusability: Once created, stored procedures can be reused in different applications without rewriting SQL code.
  • Performance Optimization: Since they are precompiled, stored procedures can execute faster than separate SQL statements.
  • Security: Stored procedures help protect the database by controlling access to data and restricting direct SQL execution.
  • Maintenance: Centralizing business logic in stored procedures makes it easier to maintain and update.

Differences Between Triggers and Stored Procedures

Feature Stored Procedures Triggers
Execution Manually invoked by CALL statement Automatically triggered by events
Use Case Perform routine tasks or calculations Enforce constraints, logging, etc.
Event Binding Not bound to specific events Bound to INSERT, UPDATE, DELETE
Return Values Can return values Cannot return values
Complexity Can include complex logic and loops Typically more straightforward

Summary:

  • Stored Procedures: Called explicitly and used for routine tasks, calculations, and business logic.
  • Triggers: Executed automatically in response to database events to enforce data integrity and audit changes.

Detailed Example: Stored Procedure for Monthly Sales Report

In a Sales Database, we want to generate a report showing total sales for a given month.

	CREATE PROCEDURE MonthlySalesReport (IN report_month INT, IN report_year INT, OUT total_sales DECIMAL(10,2))
	BEGIN
	SELECT SUM(amount) INTO total_sales
	FROM sales
	WHERE MONTH(sale_date) = report_month AND YEAR(sale_date) = report_year;
	END;

Using the Stored Procedure:

	CALL MonthlySalesReport(9, 2023, @total_sales);
	SELECT @total_sales;

Explanation:

  • IN parameters: report_month and report_year specify the month and year for the report.
  • OUT parameter: total_sales returns the calculated sales amount.

Sample Output:

@total_sales
5000.00

This stored procedure simplifies the process of generating a monthly sales report and can be reused for any month or year.
