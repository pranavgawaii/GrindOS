Advanced Views in SQL

Advanced views in SQL allow for dynamic data querying, complex data summarization, and simplified management of layered data through nested structures. They extend the capabilities of basic views by introducing aggregation, nesting, and conditions for updatability.

Querying Data from Views

Querying data from views is similar to querying data from tables. Since views are virtual tables based on SQL queries, you can retrieve data by selecting columns and applying filters or sorting criteria directly on the view.

Syntax:
SELECT columns FROM view_name WHERE condition ORDER BY column_name;

Example

Consider a view Employee_View created from the Employee table:

CREATE VIEW Employee_View AS
SELECT Employee_ID, Name, Department, Salary
FROM Employee;

You can query data from Employee_View as you would a table:

SELECT Name, Department
FROM Employee_View;

Filtering and Sorting Data from Views

To filter employees in the IT department:

SELECT Name, Salary
FROM Employee_View
WHERE Department = 'IT';

To list all employees sorted by their salary in descending order:

SELECT Name, Salary
FROM Employee_View
ORDER BY Salary DESC;

Conditions for Updating Views

Updating views in SQL has certain conditions. A view is updatable if it is based on a single table and doesn’t contain complex expressions like joins, subqueries, aggregate functions, or DISTINCT.

Requirements for Updatable Views:

  • Single Table: The view must be based on a single table.
  • Direct Mapping: Each column in the view should directly map to a column in the underlying table.
  • No Aggregations or Joins: Views with joins, aggregates, or calculated fields are typically not updatable.

Example

Consider a simple view Employee_Salary_View for updating salaries:

CREATE VIEW Employee_Salary_View AS
SELECT Employee_ID, Salary
FROM Employee;

To update a salary in this view:

UPDATE Employee_Salary_View
SET Salary = 70000
WHERE Employee_ID = 1;

This updates the salary of Employee_ID = 1 in the Employee table through the view.

Views with Aggregate Functions

Views can include aggregate functions like SUM, COUNT, AVG, MIN, and MAX to provide summarized data. These are often used to create summary views that show totals, averages, or counts.

Example

Consider an Orders table:

Order_ID Customer_ID Amount
1 101 200
2 102 150
3 101 100

To create a view that shows the total amount spent by each customer:

CREATE VIEW Customer_Summary AS
SELECT Customer_ID, SUM(Amount) AS Total_Spent
FROM Orders
GROUP BY Customer_ID;

Result of Customer_Summary View:

Customer_ID Total_Spent
101 300
102 150

Nested Views

Nested Views are views that reference other views rather than directly referencing base tables. They help build complex queries layer-by-layer and simplify data management.

Example

Consider an Employee_Department view that shows each employee with their department name:

CREATE VIEW Employee_Department AS
SELECT e.Employee_ID, e.Name, d.Department_Name
FROM Employee e
JOIN Department d ON e.Department_ID = d.Department_ID;

Now, create a High_Earners nested view to show employees with salaries above 70,000:

CREATE VIEW High_Earners AS
SELECT Employee_ID, Name, Department_Name
FROM Employee_Department
WHERE Salary > 70000;


Summary Table of SQL View Operations

Operation Description Example Use Case
Querying Views Retrieve data from a view like a table SELECT Name FROM Employee_View
Filtering Views Use WHERE to filter data SELECT Name FROM Employee_View WHERE Department='IT'
Sorting Views Use ORDER BY to sort data SELECT Name FROM Employee_View ORDER BY Salary DESC
Updating Views Modify data in views (if updatable) UPDATE Employee_View SET Salary=70000 WHERE ID=1
Aggregate Functions Use SUM, COUNT, etc., to summarize data CREATE VIEW Total_Sales AS SELECT SUM(Amount) FROM Sales
Nested Views Use a view to reference another view Create High_Earners view based on Employee_Department
