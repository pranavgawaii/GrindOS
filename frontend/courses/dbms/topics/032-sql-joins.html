Unions in SQL

In SQL, the UNION operator is used to combine the results of two or more SELECT queries into a single result set. It removes duplicate rows by default, presenting only unique results from the combined queries. The tables in each SELECT query must have the same number of columns and compatible data types for UNION to work.

Key Points of UNION

  • Combines rows from multiple SELECT queries.
  • Removes duplicates by default.
  • Requires the same number of columns and compatible data types in each query.
Syntax:
SELECT column1, column2 FROM table1
UNION
SELECT column1, column2 FROM table2;

UNION vs. UNION ALL

The UNION ALL operator is similar to UNION but includes all rows from the combined queries, including duplicates. This is useful when you want to keep all records, regardless of whether they’re repeated across the queries.

Feature UNION UNION ALL
Duplicates Removes duplicates Retains all duplicates
Use Case When unique records are needed When all records are required
Performance Slightly slower due to deduplication Faster as it skips deduplication

Example

Consider two tables:

Employee Table

Employee_ID Name
1 Alice
2 Bob

Contractor Table

Contractor_ID Name
2 Bob
3 Charlie

Using UNION:

SELECT Name FROM Employee
UNION
SELECT Name FROM Contractor;

Result (UNION):

Name
Alice
Bob
Charlie

Using UNION ALL:

SELECT Name FROM Employee
UNION ALL
SELECT Name FROM Contractor;

Result (UNION ALL):

Name
Alice
Bob
Bob
Charlie


UNION with WHERE Clause

You can use the WHERE clause in each SELECT statement within a UNION to filter data individually from each table. This allows more control over which rows are included from each query.

Example

SELECT Name FROM Employee WHERE Name = 'Bob'
UNION
SELECT Name FROM Contractor WHERE Name = 'Bob';

Result:

Name
Bob

UNION with ORDER BY Clause

The ORDER BY clause can be used with UNION to sort the combined result set. The ORDER BY clause should be applied after the final SELECT statement to sort the entire output.

Example

SELECT Name FROM Employee
UNION
SELECT Name FROM Contractor
ORDER BY Name;

Result:

Name
Alice
Bob
Charlie

UNION with GROUP BY Clause

The GROUP BY clause can be used with UNION to aggregate data across combined results. Each SELECT statement can use GROUP BY to summarize data, or GROUP BY can be applied to the final UNION result.

Example

Consider Sales and Returns tables:

Sales Table

Product_ID Quantity
101 2
102 5

Returns Table

Product_ID Quantity
101 1
103 4 
SELECT Product_ID, COUNT(*) AS Occurrences
FROM Sales
GROUP BY Product_ID
UNION
SELECT Product_ID, COUNT(*) AS Occurrences
FROM Returns
GROUP BY Product_ID;

Result:

Product_ID Occurrences
101 1
102 1
103 1

UNION with JOINs

UNION can also be combined with JOINs in SQL to merge data from multiple tables with relationships. Using JOINs with UNION allows complex queries where data is combined from multiple sources and then joined based on conditions.

Example

Consider Customer, Sales, and Returns tables:

Customer Table

Customer_ID Name
1 Alice
2 Bob

Sales Table

Customer_ID Product_ID
1 101
2 102

Returns Table

Customer_ID Product_ID
1 101
2 103 
SELECT Customer.Name, Sales.Product_ID
FROM Customer
JOIN Sales ON Customer.Customer_ID = Sales.Customer_ID
UNION
SELECT Customer.Name, Returns.Product_ID
FROM Customer
JOIN Returns ON Customer.Customer_ID = Returns.Customer_ID;

Result:

Name Product_ID
Alice 101
Bob 102
Bob 103

Summary Table of UNION Clauses

Clause Usage Description Example Outcome
UNION Combines unique records from multiple SELECTs Alice, Bob, Charlie
UNION ALL Combines all records, including duplicates Alice, Bob, Bob, Charlie
UNION with WHERE Filters data from each SELECT individually Only records matching specific criteria (e.g., Name = "Bob")
UNION with ORDER BY Sorts the combined result set Combined records sorted by specified column
UNION with GROUP BY Aggregates combined records based on a column Unique count of product occurrences
UNION with JOINs Combines and joins records from multiple tables Merged records from Sales and Returns by Customer
