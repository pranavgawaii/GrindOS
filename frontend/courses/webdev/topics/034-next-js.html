1. What is Next.js?

Next.js is a React framework that adds server-side rendering, file-based routing, API routes, and production optimizations out of the box. It's used by Vercel, Netflix, TikTok, and GitHub.

2. Rendering Strategies

StrategyWhen RenderedUse Case
CSR (Client-Side)Browser, on demandDashboards, SPA pages
SSR (Server-Side)On each requestPersonalized pages, dynamic data
SSG (Static)At build timeBlog posts, marketing pages
ISR (Incremental)Static + background revalidationE-commerce with stale-while-revalidate

3. The App Router (Next.js 13+)

app/
├── layout.tsx          ← Root layout (shared UI)
├── page.tsx            ← Home route "/"
├── about/
│   └── page.tsx        ← "/about"
├── blog/
│   ├── page.tsx        ← "/blog"
│   └── [slug]/
│       └── page.tsx    ← "/blog/my-post"
└── api/
    └── users/
        └── route.ts    ← API endpoint "/api/users"

  const post = await fetch(`https://api.example.com/posts/${params.slug}`, {
    next: { revalidate: 3600 } // revalidate every hour
  }).then(r => r.json());

{post.content}

}

// SSR — fresh on every request

{JSON.stringify(data)}
