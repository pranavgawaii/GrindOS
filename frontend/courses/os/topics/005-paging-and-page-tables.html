1. Non-Contiguous Allocation: Paging

Paging is a memory management scheme that eliminates the need for contiguous physical memory allocation. Physical memory is broken into fixed-size blocks called Frames, and logical memory is divided into blocks of the exact same size called Pages.

2. Address Translation in Paging

The CPU generates a Logical Address split into two parts:

  • Page Number (p): Used as an index into a Page Table. The page table contains the base physical frame number.
  • Page Offset (d): The specific displacement address within the page. It is concatenated with the physical frame base to form the final physical address.

3. Translation Lookaside Buffer (TLB)

Since the page table resides in RAM, every data access requires two memory cycles (one to read the page table, one to access the data). To double this performance, a high-speed hardware cache called the Translation Lookaside Buffer (TLB) is added to the Memory Management Unit (MMU) to cache recent translations.

Effective Access Time (EAT) Formula

Let h be the TLB hit ratio, m be memory access time, and c be TLB cache lookup time:

EAT = h * (c + m) + (1 - h) * (c + 2m)

4. Page Table Entry (PTE) Structure

A standard Page Table Entry contains several bit flags to manage page states:

Bit / Field Function
Frame Number The physical address base mapping.
Valid / Invalid Bit Indicates if the page is currently loaded in physical RAM.
Dirty Bit (Modify Bit) Indicates if the page has been modified since it was loaded (crucial for page replacement write-back decisions).
Referenced Bit Indicates if the page has been read/written recently (used by LRU approximation).

🎯 Topic Q&A

Q1: Why must page size and frame size be identical?

They must be identical so that the offset (`d`) in logical memory maps exactly to the offset in physical memory without mathematical translation, making address translation an instant bit-level operation.

Q2: What is the Translation Lookaside Buffer (TLB)?

The TLB is an ultra-fast associative hardware cache inside the Memory Management Unit (MMU) that stores recently accessed virtual-to-physical address mappings, preventing expensive repeated RAM lookups.

Q3: Explain the role of the Dirty Bit in a Page Table Entry.

The Dirty Bit (Modify Bit) is set to 1 when a page in memory is modified. During page replacement, if the dirty bit is 0, the OS can discard the page because it is identical to the copy on disk, avoiding expensive write-back I/O.

Q4: Calculate Effective Access Time (EAT) if memory access is 100ns, TLB search is 20ns, and TLB hit ratio is 90%.

Using the EAT formula: EAT = 0.90 * (20 + 100) + 0.10 * (20 + 200) = 108 + 22 = 130ns.

Q5: What is internal fragmentation in paging?

Internal fragmentation occurs when a process's last page is not completely filled. Because memory is allocated in fixed page sizes, if a process needs 1KB but page size is 4KB, the remaining 3KB in the frame is wasted.
