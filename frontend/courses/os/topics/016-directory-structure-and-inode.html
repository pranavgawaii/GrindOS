1. Directory Structures

A directory is a logical index file mapping user-readable file names directly to their physical identifiers (inode numbers or cluster base blocks).

Directory Layout Models

  • Single-Level Directory: All files for all users reside in a single flat directory. Simple but suffers from naming collisions.
  • Two-Level Directory: Each user has their own private User File Directory (UFD) under a root Master File Directory (MFD). Resolves collisions but prevents subfolders.
  • Tree-Structured Directory: Users can create nested subdirectories forming a tree (Standard modern model). Uses absolute and relative paths.
  • Acyclic-Graph Directory: Allows directories to share subfolders and files (links) without cycles. Supports shared access but complicates deletion (requires reference counters).

2. Inode Structure in Linux

In Unix-like systems (Ext4), an Inode (Index Node) is a hardware structure containing all information about a file except its name. The file name is stored inside directory entries mapped to the inode index.

Unix Inode Pointers

An inode contains 15 pointers to physical disk data blocks:

  • 12 Direct Pointers: Point directly to the first 12 data blocks of the file (fast access for small files).
  • 1 Single Indirect Pointer: Points to an index block containing pointers to data blocks.
  • 1 Double Indirect Pointer: Points to an index block containing pointers to single indirect blocks.
  • 1 Triple Indirect Pointer: Supports massive multi-gigabyte files using three levels of index pointers.

🎯 Topic Q&A

Q1: What is stored in a Unix directory file?

A Unix directory file does not store file data or file attributes; it simply stores a list of key-value pairs mapping human-readable filenames to their unique numerical Inode numbers.

Q2: Why does an Inode not store the filename?

To support Hard Links. By separating filenames from the inode structure, multiple directory entries can map different filenames to the identical Inode number, referencing the same physical data block.

Q3: How does the indirect pointer structure in an inode support large file sizes?

By nesting block pointer tables (single, double, triple indirection), an inode can reference exponentially larger blocks of physical disk space beyond the initial 12 direct block limitations.

Q4: Explain the difference between a Hard Link and a Soft (Symbolic) Link.

A hard link is a directory entry mapping a filename directly to an existing Inode number. A soft link (Symlink) is a separate file that stores a text path to another filename. If the original file is deleted, the hard link still works, while the soft link breaks.

Q5: What is reference counting in acyclic-graph directories?

A reference counter tracks how many hard links/directory entries point to a specific Inode. The OS only deletes a file and reclaims its disk space when the reference counter drops to 0.
