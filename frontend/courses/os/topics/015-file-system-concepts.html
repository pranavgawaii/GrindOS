1. What is a File?

A File is a contiguous logical address space mapped onto physical sectors of non-volatile secondary storage by the operating system. It represents the smallest allocation unit on disk.

File Attributes

  • Name: Human-readable string identifier.
  • Identifier: Unique integer tag (inode number) inside the file system.
  • Type: Differentiates text, executable, directory, or media files.
  • Size: Current byte capacity.
  • Protection: Read, write, execute permissions flags.

2. File Operations

The OS provides atomic system calls to manipulate files:

  • create(): Allocates space and creates a directory entry.
  • open(): Loads the file's metadata/inode into active memory.
  • read()/write(): Moves data to/from user buffers, updating the active file pointer.
  • reposition() (lseek): Updates the current file pointer index.
  • delete(): Removes directory entry, freeing disk block allocations.

3. File Access Methods

Access Method Operation Details Use Case
Sequential Access Information is processed in order, one record after another (e.g., read next block). Compilers, editors, log parsing.
Direct Access (Random) CPU can read/write blocks directly by block number index (e.g., read block 15) in O(1) time. Databases, key-value storage index queries.

🎯 Topic Q&A

Q1: What are file system access methods?

Access methods define how blocks inside a file are read and written. The two main types are: Sequential Access (processed in order) and Direct Access (any block read directly via index pointer).

Q2: Why do OS kernels require an open-file table?

Opening a file requires resolving pathnames and loading inodes from disk (slow). To optimize, the OS loads open file details into an active memory open-file table, returning a lightweight file descriptor (FD) index to the user.

Q3: Explain the difference between file metadata and file data.

File data is the actual content written by the user (text, binary). File metadata (stored in inodes) is structural information about the file (size, permissions, timestamps, owner, block pointers).

Q4: What is the purpose of the lseek system call?

The `lseek()` system call repositions the read/write file pointer offset in the open-file table without reading or writing data, enabling random access within a file.

Q5: Can you write a Python snippet reading a file in blocks? 
# Read file in 1024-byte chunks
def read_chunks(file_path):
    with open(file_path, 'rb') as f:
        while True:
            chunk = f.read(1024)
            if not chunk:
                break
            yield chunk
