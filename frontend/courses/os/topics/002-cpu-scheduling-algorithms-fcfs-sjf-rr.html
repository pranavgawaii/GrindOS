1. CPU Scheduling Overview

CPU scheduling is the process by which the operating system decides which process in the ready queue receives CPU core cycles. It is divided into two primary modes:

  • Preemptive Scheduling: The OS can interrupt a running process to assign the CPU to another process (e.g., when a higher-priority process arrives or a time slice expires).
  • Non-Preemptive Scheduling: Once a process is allocated the CPU, it holds it until it voluntary terminates or blocks for I/O.

2. Core Scheduling Algorithms

A. First-Come, First-Served (FCFS)

Non-preemptive scheduling based on arrival order. Suffer from the **Convoy Effect**, where short processes are blocked behind a massive, long-running process, ballooning the average waiting time.

B. Shortest Job First (SJF)

Selects the process with the shortest CPU burst time next. It is mathematically optimal for minimizing average waiting time. Its preemptive variant is called **Shortest Remaining Time First (SRTF)**. Its primary drawback is **Starvation** of longer jobs.

C. Round Robin (RR)

Preemptive, time-sliced algorithm designed for interactive systems. Each process runs for a small, fixed unit of time called a **Time Quantum (TQ)**. If a process does not complete within the TQ, it is preempted and returned to the end of the ready queue.

D. Priority Scheduling

Each process is assigned a priority integer. The CPU is allocated to the highest-priority process. To prevent starvation of low-priority processes, **Aging** is used (gradually increasing the priority of processes as they wait).

3. Numerical Trace & Gantt Chart Example

Consider 3 processes arriving at time 0 with the following burst times:

Process Burst Time Priority (Lower = Higher)
P1 24 3
P2 3 1
P3 3 2

Gantt Chart for FCFS (P1 → P2 → P3)

P1 (24ms) P2 (3ms) P3 (3ms)
0 24 27 30
  • Waiting Time: P1 = 0, P2 = 24, P3 = 27.
  • Average Waiting Time: 17.0ms.

Gantt Chart for SJF / Priority (P2 → P3 → P1)

P2 (3ms) P3 (3ms) P1 (24ms)
0 3 6 30
  • Waiting Time: P1 = 6, P2 = 0, P3 = 3.
  • Average Waiting Time: 3.0ms! (Mathematically optimal reduction).

🎯 Topic Q&A

Q1: What is the Convoy Effect in FCFS scheduling?

The Convoy Effect is a phenomenon where several short processes wait in the ready queue behind one long-running, CPU-bound process. This severely degrades CPU utilization and average waiting time, as all other resources remain idle waiting for the giant job to finish.

Q2: How does Round Robin perform under different time quantum lengths?

If the time quantum is too long, Round Robin behaves exactly like FCFS. If the quantum is too short, the CPU spends too much time context switching instead of executing useful code, creating massive system overhead.

Q3: What is Starvation in CPU scheduling and how is it resolved?

Starvation occurs when low-priority or long burst-time processes wait indefinitely in the queue because high-priority or shorter jobs keep arriving. It is resolved using Aging, which gradually increases the priority of waiting processes over time.

Q4: Why is Shortest Job First (SJF) considered optimal?

SJF is optimal because it schedules the shortest CPU burst first. This mathematically minimizes the average waiting time for any given set of processes by moving long-waiting processes to the back of the queue.

Q5: What is the difference between preemptive and non-preemptive scheduling?

In non-preemptive scheduling, a process retains CPU ownership until it voluntary terminates or blocks for I/O. In preemptive scheduling, the OS can forcefully interrupt a running process to allocate the CPU to another thread (e.g., when a time slice ends or higher-priority thread arrives).
