1. Page Replacement Overview

When physical memory (RAM) is completely full and a page fault occurs, the operating system must select an existing frame (a victim page) to overwrite. This choice is determined by a **Page Replacement Algorithm** with the goal of minimizing page faults.

2. Core Algorithms

A. First-In, First-Out (FIFO)

Replaces the oldest page in memory. While simple, it suffers from **Belady's Anomaly**, where allocating more physical frames paradoxically increases the page fault rate for certain reference strings.

B. Optimal Page Replacement (OPT)

Replaces the page that will not be accessed for the longest period in the future. It is a theoretical algorithm that guarantees the lowest possible page fault rate, but **cannot be implemented** because future memory references are unknown.

C. Least Recently Used (LRU)

Replaces the page that has not been accessed for the longest duration of time, leveraging temporal locality history. It is highly performant and immune to Belady's Anomaly but requires hardware support (counters/stacks).

3. Numerical Trace Example

Given 3 physical frames, trace reference string: 7, 0, 1, 2, 0, 3, 0, 4:

FIFO Trace (7 Page Faults)

Ref 7 0 1 2 0 3 0 4
Frame 1 7 7 7 2 2 2 0 0
Frame 2 - 0 0 0 0 3 3 3
Frame 3 - - 1 1 1 1 1 4

LRU Trace (6 Page Faults)

Ref 7 0 1 2 0 3 0 4
Frame 1 7 7 7 2 2 2 2 4
Frame 2 - 0 0 0 0 0 0 0
Frame 3 - - 1 1 1 3 3 3

🎯 Topic Q&A

Q1: What is Belady's Anomaly?

Belady's Anomaly is a phenomenon where the page fault rate increases when more physical memory frames are allocated to a process. It occurs in first-in, first-out (FIFO) page replacement algorithms but does not happen in stack-based algorithms like LRU.

Q2: Why cannot the Optimal Page Replacement algorithm be implemented in practice?

Because it requires perfect future knowledge of all memory reference strings that the CPU will execute, which is impossible to predict dynamically in a multitasking OS.

Q3: How is LRU implemented in hardware?

It is typically implemented in two ways: 1) Counters: associating each page entry with a register that updates to the CPU clock on access. 2) Stacks: maintaining a hardware stack of page references, moving accessed pages to the top of the stack.

Q4: Explain the LRU Approximation Second Chance (Clock) algorithm.

Since true LRU is expensive, the Clock algorithm uses a single **Referenced Bit**. It arranges frames in a circular queue. When a victim is needed, the clock hand scans: if referenced is 1, it sets it to 0 and moves on (giving a second chance); if referenced is 0, it replaces that page.

Q5: Can you write a Python function tracing a simple LRU cache operation? 
# Simple LRU Cache simulation outline
def lru_refer(cache, capacity, page):
    if page in cache:
        cache.remove(page)
    elif len(cache) >= capacity:
        cache.pop(0) # Evict LRU (index 0)
    cache.append(page) # Make MRU
