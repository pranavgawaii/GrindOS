1. Mechanical Disk Latency Components

Accessing data on a mechanical Hard Disk Drive (HDD) involves physical seek operations:

  • Seek Time: The time required for the mechanical arm to move to the correct cylinder track. **Seek time is the most expensive access delay.**
  • Rotational Latency: The time spent waiting for the target sector to rotate under the read/write head.

2. Disk Scheduling Algorithms

To minimize head movement (seek time), the OS schedules pending I/O block requests in an optimal order.

A. First-Come, First-Served (FCFS)

Processes requests sequentially as they arrive. Simple, but results in high head seek times because it does not optimize head coordinates.

B. Shortest Seek Time First (SSTF)

Selects the request closest to the current head position. While fast, it can cause **Starvation** of distant requests if new close requests keep arriving.

C. SCAN (Elevator Algorithm)

The head sweeps continuously from one end of the disk to the other end, servicing all requests, and then reverses direction. Prevents starvation.

D. C-SCAN (Circular SCAN)

The head sweeps from one end to the other, but when it reaches the boundary, it returns to the starting boundary instantly without servicing requests on the return sweep, providing a more uniform waiting time.

E. LOOK and C-LOOK

Optimized variants of SCAN and C-SCAN. Instead of forcing the head to travel to the absolute outer boundary of the disk volume, the head only travels as far as the outermost pending request before reversing.

3. Numerical Trace Example

Consider a disk queue with requests at cylinders: 98, 183, 37, 122, 14, 124, 65, 67, with head starting at 53:

Algorithm Order of Service Total Head Seek Distance
FCFS 53 → 98 → 183 → 37 → 122 → 14 → 124 → 65 → 67 640 cylinders
SSTF 53 → 65 → 67 → 37 → 14 → 98 → 122 → 124 → 183 236 cylinders

🎯 Topic Q&A

Q1: Which component of disk access latency is the most expensive?

Seek Time (the time taken to mechanically reposition the disk arm onto the correct cylinder track) is the most expensive disk access delay.

Q2: Why does SSTF suffer from process starvation?

Because it constantly selects the request closest to the current head location. If new requests keep arriving close to the head, distant requests will wait indefinitely, starving of I/O resources.

Q3: How does C-SCAN improve upon the standard SCAN algorithm?

C-SCAN provides more uniform waiting times. In standard SCAN, cylinders near the center are scanned twice as often as edge cylinders. C-SCAN sweeps in only one direction and jumps back, offering a fair waiting time for all disk sectors.

Q4: Explain the difference between SCAN and LOOK scheduling.

SCAN forces the read/write head to travel to the absolute boundary of the disk volume before reversing. LOOK checks if there are pending requests ahead; if no more requests exist in the active sweep direction, it reverses immediately, avoiding useless travel.

Q5: Trace the SSTF seek calculations from 53 to 65.

Initial position is 53. The distances are: |53-65|=12, |53-37|=16. The closest is 65, so head moves to 65. From 65, distances are: |65-67|=2, |65-37|=28. Closest is 67. The sequence continues, yielding 236 total seek distance.
