1. Concept of Segmentation

Unlike paging, which is a physical address abstraction invisible to programs, Segmentation is a memory management scheme that aligns with the programmer's logical view of memory. A program is divided into logical, variable-sized segments (e.g., Code, Stack, Heap, Global Variables, Standard Libraries).

2. Segment Address Translation

The CPU generates a logical address containing two parts: <segment-number (s), offset (d)>. The **Segment Table** contains two values for each segment:

  • Base: Starting physical address of the segment in RAM.
  • Limit: The exact physical length of the segment.

The Translation Rule

The hardware MMU checks if offset (d) < limit. If it is valid, the physical address is calculated as: Physical Address = Base + Offset (d). If the offset exceeds the limit, a **Segment Fault (Trap)** is thrown, protecting memory from out-of-bounds corruption.

3. Paging vs. Segmentation Comparison

Dimension Paging Segmentation
Block Size Fixed-sized blocks (Pages/Frames). Variable-sized blocks (Segments).
Fragmentation Suffers from **Internal Fragmentation**. Suffers from **External Fragmentation** (free slots between segments).
User Visibility Invisible to the programmer (managed entirely by hardware/OS). Visible to the compiler and programmer as logical modules.
Address Form Single flat address mapped by page indices. Two-dimensional address: <segment, offset>.
Sharing & Security Harder to share logical modules because they span arbitrary pages. Extremely easy to secure or share (e.g., mark the Code segment as Read-Only).

🎯 Topic Q&A

Q1: What is the main difference in how physical addresses are computed in paging vs. segmentation?

In paging, the physical address is formed by concatenating the frame number and the offset. In segmentation, the physical address is calculated by adding the segment base address to the offset, requiring addition operations.

Q2: Why does segmentation suffer from external fragmentation?

Because segments are of variable sizes and are loaded/unloaded dynamically, RAM becomes partitioned into alternating free and occupied slots. Over time, the sum of free slots might exceed a request, but we cannot allocate it because it is not contiguous.

Q3: How does the MMU validate safety in segmentation?

The MMU checks if the logical offset (`d`) is strictly less than the segment's limit. If the offset is larger, the CPU throws a hardware exception (trap), preventing buffer overflow security exploits.

Q4: Why is it easier to implement protection and sharing in segmentation?

Since segments correspond directly to logical code modules (e.g., read-only code, read-write data), protection flags (read, write, execute) can be applied to the entire segment uniformly, whereas in paging, logical modules are scattered randomly across pages.

Q5: What is segmented paging (hierarchical translation)?

To combine the benefits of both worlds, modern architectures page the segments. A program is divided into logical segments, and each segment is subdivided into fixed-size pages, eliminating external fragmentation while preserving logical modules.
