1. What is Object-Oriented Programming (OOP)?

Object-Oriented Programming (OOP) is a software programming paradigm that structures code around **Objects** (data entities) rather than actions or logic. An object is an active instance of a blueprint class, holding attributes (data/state) and methods (behaviors).

2. OOP vs. Procedural Programming

Dimension Procedural Programming (e.g., C, Pascal) Object-Oriented Programming (e.g., Python, C++, Java)
Approach Top-Down approach. Focuses on functions and step-by-step procedures. Bottom-Up approach. Focuses on real-world data entities (objects) and models.
Security Low security. Data moves freely across global scopes and functions. High security. Data is hidden inside objects (encapsulation) and accessed via controlled methods.
Code Reusability Low. Relies heavily on copying/modifying raw code functions. High. Encouraged through inheritance, polymorphism, and composition.
Modularity Hard to scale. Large codebases are difficult to manage and debug. Easy to scale. Objects are isolated, modular, and self-contained units.

3. The Four Pillars of OOP

The core of OOP rests on four fundamental pillars:

  1. Encapsulation: Bundling data and methods into a single class block, restricting direct external access to variables.
  2. Inheritance: Allowing a new class (subclass) to inherit fields and methods from an existing class (parent), reducing duplication.
  3. Polymorphism: Allowing different objects to respond to the same method call signature in unique, class-specific ways.
  4. Abstraction: Hiding complex inner execution details, exposing only essential, clean interfaces.

🎯 Topic Q&A

Q1: Why was the OOP paradigm introduced?

OOP was introduced to solve the complexity and un-maintainability of large procedural codebases. By grouping data and behaviors into self-contained objects, OOP offers modularity, data safety, and reusability.

Q2: What is the differences between top-down and bottom-up programming?

Top-down (procedural) breaks a problem into subtasks and writes sequential functions. Bottom-up (OOP) builds independent, reusable models of data and actions first, and aggregates them to solve the global problem.

Q3: What are the 4 main pillars of OOP?

They are: 1) Encapsulation. 2) Inheritance. 3) Polymorphism. 4) Abstraction.

Q4: Can a class be created without an object?

Yes. A class is a structural definition (blueprint) stored in code. It does not occupy active memory for fields until you instantiate it to create an object.

Q5: What is an Object in terms of memory?

An object is an active memory allocation in the heap containing the instance variable states, metadata pointers linking to its Class definition, and references to its method blocks.
