1. What is the Factory Method Pattern?

The Factory Method Pattern is a creational pattern that defines an interface for creating objects in a superclass, but allows subclasses to alter the type of objects created. It decouples the object creation logic from client code.

2. Python Factory Implementation

We define abstract interfaces and instantiate concrete products dynamically:

from abc import ABC, abstractmethod

# Product Interface
class Animal(ABC):
    @abstractmethod
    def speak(self): pass

# Concrete Products
class Dog(Animal):
    def speak(self): return "Woof!"
class Cat(Animal):
    def speak(self): return "Meow!"

# Creator (Factory)
class AnimalFactory:
    @staticmethod
    def get_animal(animal_type):
        if animal_type == "dog":
            return Dog()
        elif animal_type == "cat":
            return Cat()
        raise ValueError("Invalid animal type")

pet = AnimalFactory.get_animal("dog")
print(pet.speak())

🎯 Topic Q&A

Q1: What are the main benefits of the Factory Design Pattern?

It provides **Loose Coupling** by separating object creation logic from client usage, respects the **Single Responsibility Principle**, and is open for extensions without editing client files.

Q2: How does the Factory Pattern support the Open/Closed Principle?

If we need to add a new product (e.g., `Parrot`), we create the subclass and update the factory dispatcher without changing any existing client code that consumes `Animal` interfaces.

Q3: Contrast Factory Method and Abstract Factory.

Factory Method uses a single function to create one product class. Abstract Factory is a parent factory that creates families of related dependent products (e.g., creating a dark-theme Button and a dark-theme Scrollbar together).

Q4: When should you avoid using a Factory Pattern?

For simple class models. Introducing factories to instantiate standard, stable objects adds unnecessary cognitive overhead and boilerplate classes to a codebase.

Q5: Can you write a Python Factory returning different DB connections dynamically? 
class DBFactory:
    @staticmethod
    def connect(db_type):
        if db_type == "postgres": return PostgresConn()
        if db_type == "mongo": return MongoConn()
        raise ValueError("Unsupported DB")
