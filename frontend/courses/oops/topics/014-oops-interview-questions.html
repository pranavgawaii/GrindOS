20 Most Common OOPs Placement Questions

Prepare model answers for key concepts:

  1. What is the difference between abstraction and encapsulation?
    • *Answer:* Abstraction is the process of hiding complex design logic to simplify interfaces (design level). Encapsulation is the process of bundling data and methods inside a class to restrict direct access (implementation level).
  2. Does Python support multiple inheritance? How does it resolve conflicts?
    • *Answer:* Yes, Python supports multiple inheritance. It resolves method name collisions using **Method Resolution Order (MRO)** powered by the **C3 Linearization** algorithm.
  3. Explain the SOLID design principles.
    • *Answer:* Single Responsibility, Open/Closed (extend, don't modify), Liskov Substitution (subclass safety), Interface Segregation, and Dependency Inversion (depend on abstractions).
  4. What is dynamic binding (runtime polymorphism)?
    • *Answer:* Resolving a method call during program execution based on the dynamic type of the object, rather than compiling it statically (e.g., method overriding).

🎯 Topic Q&A

Q1: What is the difference between a class and an interface?

A class is a structural template defining both attributes and concrete code implementations. An interface contains only abstract method signatures without code execution details, enforcing a strict contract on subclasses.

Q2: What is the Liskov Substitution Principle?

It states that subtypes must be completely substitutable for their parent types without altering program correctness, preventing unexpected errors during runtime polymorphism.

Q3: Contrast deep copy and shallow copy in Python.

Shallow copy duplicates the outer object container but shares references to inner nested mutable elements. Deep copy recursively duplicates all elements, creating completely independent object trees.

Q4: Explain the difference between Composition and Inheritance.

Inheritance is an "Is-A" relationship creating tight coupling. Composition is a "Has-A" relationship that nests component objects inside a class, providing loose coupling and dynamic modifications.

Q5: What are magic/dunder methods in Python?

They are special hooks starting and ending with double underscores (e.g., `__init__`, `__str__`) that let you customize core operator behaviors for user-defined object classes.
