1. What are Dunder Methods?

Dunder Methods (Double Underscore / Magic Methods) are special hooks in Python starting and ending with `__`. They allow you to customize how your custom objects interact with native operators (e.g., `+`, `==`, `len()`).

2. Core Dunder Protocols

  • `__str__` vs. `__repr__`:
    • `__str__`: Returns an aesthetic, user-friendly string representation (`print(obj)`).
    • `__repr__`: Returns a detailed, unambiguous developer representation (`repr(obj)`).
  • `__len__`: Allows using `len(obj)`.
  • `__eq__`: Overloads the comparisons `==` operator.
  • `__add__`: Overloads the mathematical `+` addition operator.

3. Operator Overloading Example

Let's design a custom 2D Vector that overloads the addition operator:

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
        
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"
        
    def __add__(self, other):
        # Overloads vector1 + vector2
        return Vector(self.x + other.x, self.y + other.y)

v1 = Vector(2, 4)
v2 = Vector(5, 3)
print(v1 + v2) # Output: Vector(7, 7)

🎯 Topic Q&A

Q1: What is the fundamental difference between `__str__` and `__repr__`?

`__str__` returns a friendly, readable string targeting end-users. `__repr__` returns an unambiguous, detailed string targeting developers, often displaying exact code representation to recreate the object.

Q2: How does Python evaluate the `+` operator on custom classes?

When executing `a + b`, Python checks if the class of `a` defines the `__add__` dunder method. If it does, it calls `a.__add__(b)`. If not, it falls back to checking `b` for `__radd__` before throwing a TypeError.

Q3: How would you make an object check membership using the `in` operator?

You overload the `__contains__(self, item)` dunder method, which returns `True` or `False` depending on item presence.

Q4: Why are dunder methods also called "magic" methods?

Because you rarely invoke them directly (e.g., calling `obj.__init__()`). Instead, they are implicitly triggered behind the scenes by native Python operations and operators.

Q5: Write a Python class implementing `__len__` and `__eq__` comparisons. 
class Group:
    def __init__(self, members):
        self.members = members
    def __len__(self):
        return len(self.members)
    def __eq__(self, other):
        return self.members == other.members
