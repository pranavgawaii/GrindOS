1. What is Polymorphism?

Polymorphism (Greek for "many forms") is the ability of different object classes to respond to the identical method signature call in unique ways, making APIs uniform.

2. Method Overriding vs. Method Overloading

Dimension Method Overriding (Runtime) Method Overloading (Compile-time)
Definition A child class redefines a parent method with the identical signature. Defining multiple methods in a class with the same name but different parameters.
Python Support Fully supported. Native child mapping resolves this at runtime. **Not supported natively**. If defined twice, Python only recognizes the latest definition.
Python Workaround Use normal inheritance overriding, optionally using the `@override` decorator in Python 3.12. Achieved using **Default Arguments** or variable argument pointers (`*args`, `**kwargs`).

Overloading Workaround in Python

class Calculator:
    def add(self, a, b, c=0): # Handle variable parameter lists
        return a + b + c

3. Duck Typing

Python utilizes Duck Typing (a form of dynamic polymorphism). Scopes determine object compatibility based on the methods they contain, rather than explicit parent inheritance hierarchies:

"If it walks like a duck and quacks like a duck, it is a duck." 
class Duck:
    def fly(self): print("Flap flap")
class Airplane:
    def fly(self): print("Jet engines roaring")

def let_it_fly(obj):
    obj.fly() # Dynamic polymorphism; works for both without inheritance!

🎯 Topic Q&A

Q1: Why doesn't Python support traditional method overloading?

Because Python is a dynamically typed language. Variable declarations don't require static type definitions, and method routing is done dynamically by resolving the method name inside the object's dictionary namespace (`__dict__`).

Q2: How do we simulate method overloading in Python?

We simulate it by using default parameter values (e.g., `c=None`) or by accepting variable argument tuples/dictionaries (`*args`, `**kwargs`) and parsing their contents inside the method.

Q3: Explain the concept of Duck Typing.

Duck typing is a dynamic polymorphism concept where an object's suitability is determined by the presence of specific methods (its interface), rather than its explicit class inheritance hierarchy.

Q4: What is the difference between compile-time and runtime polymorphism?

Compile-time polymorphism (overloading) resolves method routing during code compilation. Runtime polymorphism (overriding) resolves method calls during execution based on the dynamic type of the object.

Q5: Write a Python function showing how `*args` handles variable inputs. 
def sum_all(*args):
    return sum(args)
print(sum_all(1, 2))    # 3
print(sum_all(1, 2, 3)) # 6
