1. Structuring Object Relationships

Design patterns organize relationships between classes in two primary ways:

A. Inheritance ("Is-A" relationship)

A class inherits attributes and methods directly from a base class.

  • *Example:* `Car` is a `Vehicle`. `Manager` is an `Employee`.
  • Tradeoff: High coupling. Child code is fragile; any updates to base class classes cascade down, risking breaking subclass invariants.

B. Composition ("Has-A" relationship)

A class holds instance variables referencing other helper classes as components.

  • *Example:* `Car` **has an** `Engine` component. `User` **has a** `Profile` component.
  • Tradeoff: Loose coupling. You can swap component classes dynamically at runtime, ensuring modularity.

2. Composition Implementation Example

class Engine:
    def start(self): return "Vroom vroom"

class Car:
    def __init__(self):
        self.engine = Engine() # Composition
        
    def start(self):
        return self.engine.start() # Delegates behaviour

my_car = Car()
print(my_car.start())

🎯 Topic Q&A

Q1: What does the design principle "Favor Composition over Inheritance" mean?

It means that systems should prioritize building complex behaviors by combining simple, modular component objects (Composition) rather than creating deep, tightly-coupled class hierarchies (Inheritance) that are difficult to refactor.

Q2: Contrast "Is-A" and "Has-A" relationships with real-world examples.

"Is-A" (Inheritance) models type hierarchies, e.g., a `Dog` is an `Animal`. "Has-A" (Composition) models component inclusion, e.g., a `Computer` has a `CPU` and a `HardDrive` class.

Q3: Why is deep inheritance considered fragile?

Because subclasses are tightly coupled to parent classes. If a developer alters a parent class method or attribute signature, it can inadvertently break execution paths or variable naming inside distant subclasses.

Q4: How does composition enable dynamic, runtime behavior modifications?

Since composition uses standard instance variables, you can swap component objects at runtime (e.g., switching a `Car.engine` from `PetrolEngine()` to `ElectricEngine()`), which is impossible under static class inheritance structures.

Q5: Can composition and inheritance be combined?

Yes. Concrete subclasses can inherit from abstract base classes while using composition internally to assign helper components, combining structural standardization with modular execution.
