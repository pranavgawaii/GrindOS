1. What is Inheritance?

Inheritance is a mechanism in OOP where a new class (derived/child class) inherits attributes and methods from an existing class (base/parent class), facilitating code reusability.

2. Types of Inheritance in Python

  • Single Inheritance: A child inherits from exactly one parent.
  • Multilevel Inheritance: A child inherits from a parent, which inherits from a grandparent.
  • Hierarchical Inheritance: Multiple child classes inherit from the same parent.
  • Multiple Inheritance: A child inherits from more than one parent.
class ParentA:
    def show_a(self): print("A")
class ParentB:
    def show_b(self): print("B")
class Child(ParentA, ParentB): # Multiple Inheritance
    pass

3. The super() Function

The `super()` function returns a proxy object that delegates method calls to a parent or sibling class, allowing constructors or methods to extend parent logic instead of overwriting it:

class Employee:
    def __init__(self, name, id):
        self.name = name
        self.id = id
        
class Manager(Employee):
    def __init__(self, name, id, team_size):
        super().__init__(name, id) # Extends Employee constructor
        self.team_size = team_size

4. Method Resolution Order (MRO)

In multiple inheritance, Python resolves name conflicts (the Diamond Problem) using a strict sequence called the Method Resolution Order (MRO), which is calculated using the **C3 Linearization Algorithm**.

  • MRO guarantees: Subclasses are checked before parents, and left-listed parents before right-listed parents.
  • You can inspect MRO sequence via ClassName.__mro__ or ClassName.mro().

🎯 Topic Q&A

Q1: What is the Diamond Problem and how does Python resolve it?

The Diamond Problem is an ambiguity that arises in multiple inheritance when a child inherits from parents A and B, which both inherit from a single grandparent G. If A and B override a grandparent method, which one does the child execute? Python resolves this using Method Resolution Order (MRO) and C3 Linearization.

Q2: Why do we use `super()` inside a child constructor?

We use `super()` to invoke the parent class's constructor, ensuring that all base attributes and initialization setups are executed properly, allowing the child class to extend base logic cleanly.

Q3: How can you check the Method Resolution Order of a class?

You can check it by inspecting the `__mro__` attribute (e.g., `Child.__mro__`) or by calling the `mro()` method on the class.

Q4: Explain the difference between single and multiple inheritance.

Single inheritance allows a subclass to inherit from a single parent class. Multiple inheritance allows a subclass to inherit directly from more than one parent class (e.g., `class Child(ParentA, ParentB)`).

Q5: Write a simple Python example showcasing MRO ordering in multiple inheritance. 
class X: pass
class Y: pass
class Z(X, Y): pass
print(Z.__mro__) # Output: (Z, X, Y, object)
