1. Classes and Objects in Python

A Class is a user-defined blueprint or template that defines variables and behaviors. An Object is a physical instance of that class allocated in memory.

Syntax and Instance Instantiation

class Student:
    # Class Variable (Shared across all instances)
    university = "MITADT Pune"
    
    # Constructor (Initialiser)
    def __init__(self, name, cgpa):
        self.name = name       # Instance Variable
        self.cgpa = cgpa       # Instance Variable
        
    def display(self):
        return f"{self.name} has CGPA {self.cgpa} at {self.university}"

# Instantiating Objects
s1 = Student("Pranav", 9.5)
s2 = Student("Amit", 8.9)
print(s1.display())

2. Understanding self and __init__

  • __init__ method: Python's constructor, invoked automatically when `Student("Pranav", 9.5)` is called. It initializes instance state.
  • self pointer: References the specific active object instance currently calling the method. It allows Python to route variable actions to the correct memory block.

3. Instance vs. Class Variables

Dimension Instance Variable Class Variable (Static)
Definition Declared inside methods (usually `__init__`) prefixed with `self`. Declared directly inside the class body, outside any methods.
Memory Location Each object has its own separate memory copy. One single copy shared in class metadata memory.
Access Method Accessed via `self.variable` or `instance_name.variable`. Accessed via `ClassName.variable` or `self.variable`.

🎯 Topic Q&A

Q1: What is the purpose of the `self` parameter in Python methods?

`self` represents the active instance of the object currently executing the method. It allows Python to locate and access the correct instance variables stored in the object's specific memory scope.

Q2: Contrast instance variables and class variables in Python.

Instance variables are declared inside `__init__` via `self` and hold states unique to each object. Class variables are declared inside the class body and are shared by all instances of that class in memory.

Q3: What happens if you modify a class variable using `self.variable = new_value`?

Instead of modifying the shared class variable, Python creates a new **instance variable** with that name shadowing the class variable for that specific object only. To modify a class variable, access it via ClassName.

Q4: Why does Python require an explicit `self` parameter, unlike Java's implicit `this`?

Explicit `self` aligns with Python's design philosophy: "Explicit is better than implicit." It simplifies dynamic method additions to classes at runtime and keeps scoping rules simple.

Q5: Can you define a class in Python with zero attributes and methods?

Yes. You can use the `pass` placeholder: class Empty: pass. You can still instantiate it and add dynamic attributes to it at runtime.
