1. What is the Singleton Pattern?

The Singleton Pattern is a creational design pattern that restricts the instantiation of a class to exactly one globally shared instance, providing a unified access point (e.g., database connection pool, logger registry).

2. Python Singleton Implementation

We implement a Singleton in Python by overriding the allocator magic method `__new__`:

class Singleton:
    _instance = None
    
    def __new__(cls, *args, **kwargs):
        if not cls._instance:
            # Allocates and caches instance
            cls._instance = super(Singleton, cls).__new__(cls)
        return cls._instance

s1 = Singleton()
s2 = Singleton()
print(s1 is s2) # Output: True (Identical memory block)

3. Thread-Safe Singleton

In multi-threaded environments, two threads could concurrently execute the `if not cls._instance` check, creating two instances. We prevent this using a synchronization lock and **Double-Checked Locking**:

import threading

class ThreadSafeSingleton:
    _instance = None
    _lock = threading.Lock()
    
    def __new__(cls):
        if not cls._instance: # First check (No lock overhead if created)
            with cls._lock:
                if not cls._instance: # Second check (Thread safe)
                    cls._instance = super(ThreadSafeSingleton, cls).__new__(cls)
        return cls._instance

🎯 Topic Q&A

Q1: What is the Singleton Design Pattern and when should it be used?

A Singleton restricts a class to one single active instance. It should be used to manage global shared resources like logger registries, database connection pools, or application state configurations.

Q2: Why is the standard `__new__` singleton implementation unsafe in multi-threaded systems?

Because of race conditions. If two threads execute `__new__` at the same time before `_instance` is set, both checks pass, creating two distinct object allocations in memory.

Q3: Explain Double-Checked Locking in Singletons.

Double-checked locking minimizes lock overhead. It checks if `_instance` is empty before locking. If empty, it acquires the thread lock and checks `_instance` again, ensuring only the first thread performs the instantiation while keeping subsequent fetches extremely fast.

Q4: What is an alternative way to implement a Singleton in Python besides `__new__`?

Using a **Module-level instance**. Python modules are cached on first import. By instantiating an object at the module level and importing that instance, Python natively enforces a Singleton structure.

Q5: Can you inherit from a Singleton class?

It is difficult and highly discouraged. Subclassing singletons complicates MRO constructor logic and can inadvertently trigger `__new__` clashes, corrupting base class caches.
