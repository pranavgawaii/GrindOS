1. What is Abstraction?

Abstraction is the process of hiding complex inner implementation details and showing only the essential interfaces to the outer world. It separates *what* a class does from *how* it does it, reducing cognitive overhead.

2. Abstract Classes in Python (abc Module)

In Python, abstraction is enforced using the `abc` module. An **Abstract Class** is a class that cannot be instantiated directly and must contain at least one **Abstract Method**.

from abc import ABC, abstractmethod

class PaymentGateway(ABC): # Inherits from ABC
    @abstractmethod
    def process_payment(self, amount):
        pass

# pg = PaymentGateway() # Throws TypeError: Can't instantiate!

3. Enforcing Interfaces

Any concrete subclass inheriting from an abstract class **must** override all its abstract methods before it can be instantiated. This guarantees a strict contract:

class StripePayment(PaymentGateway):
    def process_payment(self, amount):
        print(f"Stripe processing ${amount}") # Instantiation allowed!

4. Abstract Class vs. Concrete Class

Dimension Abstract Class (using ABC) Concrete Class (Standard Class)
Instantiation Cannot be instantiated directly via constructor. Can be instantiated directly.
Abstract Methods Must contain at least one `@abstractmethod`. Cannot contain abstract methods; all methods must have code definitions.
Purpose Acts as a design blueprint and enforces contracts across sibling classes. Acts as a functional implementation of objects and attributes.

🎯 Topic Q&A

Q1: What is an abstract class and how is it defined in Python?

An abstract class is a template class that cannot be instantiated directly. In Python, it is defined by inheriting from the base `ABC` class in the `abc` module and marking methods with the `@abstractmethod` decorator.

Q2: What happens if a concrete subclass fails to implement an abstract method?

The compiler/runtime will throw a `TypeError: Can't instantiate abstract class [Subclass] with abstract method...` when you attempt to instantiate the subclass object.

Q3: Contrast an Interface and an Abstract Class.

An Interface only declares method signatures without any implementation. An Abstract class can declare both abstract signatures *and* concrete helper methods/fields that are shared by subclasses.

Q4: Why does Python not have an explicit `interface` keyword like Java?

Because Python leverages Duck Typing. Interface enforcement is implicitly handled by method presence at runtime, and formal contracts are managed elegantly using abstract base classes (ABCs).

Q5: Can an abstract class in Python have a constructor (__init__)?

Yes. An abstract class can have a constructor to initialize base attributes, which child classes can invoke using `super().__init__()` during their instantiation sequence.
