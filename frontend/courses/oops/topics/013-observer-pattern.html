1. What is the Observer Pattern?

The Observer Pattern (Pub-Sub) is a behavioral pattern defining a one-to-many relationship: when one object (the **Subject/Publisher**) modifies its state, it automatically notifies and updates all its registered dependents (**Observers/Subscribers**).

2. Python Observer Implementation

class Publisher:
    def __init__(self):
        self._subscribers = [] # Active Observers list
        
    def register(self, sub):
        self._subscribers.append(sub)
        
    def notify(self, message):
        for sub in self._subscribers:
            sub.update(message)

class Subscriber:
    def __init__(self, name):
        self.name = name
    def update(self, message):
        print(f"{self.name} notified: {message}")

pub = Publisher()
s1 = Subscriber("Alice")
pub.register(s1)
pub.notify("System update at 10 PM")

🎯 Topic Q&A

Q1: What problem does the Observer Pattern solve?

It solves the challenge of synchronizing states between multiple objects without creating tight coupling or requiring polling loops (repeated queries), establishing event-driven notifications.

Q2: How does the Observer Pattern decoupling work?

The Subject/Publisher only knows that observers implement a specific `update()` interface. It does not need to know the inner workings of concrete subscriber classes, allowing you to add subscribers at runtime.

Q3: Explain the difference between Push and Pull models in the Observer pattern.

In the **Push model**, the Subject sends detailed state updates directly to observers. In the **Pull model**, the Subject only sends a notification alert, and observers query the Subject's API to fetch the specific data they need.

Q4: What is the main risk when using the Observer Pattern?

**Memory Leaks (Lapsed Listener Problem)**. If a subscriber is deleted by the client but forgets to deregister from the Subject, the Subject's active list retains a reference, preventing Garbage Collection.

Q5: Can you write a Python un-register method for the Publisher? 
def unregister(self, sub):
    if sub in self._subscribers:
        self._subscribers.remove(sub)
