1. Constructor Basics in Python

A constructor is a special method automatically called when an object is instantiated. Python supports two levels of object creation:

__init__ vs. __new__

  • `__new__(cls, *args, **kwargs)`: The actual **creator**. It is a static method that allocates memory and returns a brand new object instance.
  • `__init__(self, *args, **kwargs)`: The **initialiser**. It receives the instance created by `__new__` and configures instance variables.
class Creator:
    def __new__(cls, *args, **kwargs):
        print("1. Allocating memory via __new__")
        instance = super(Creator, cls).__new__(cls)
        return instance
        
    def __init__(self):
        print("2. Initialising state via __init__")

obj = Creator()

2. Destructor: __del__

The `__del__` method acts as a destructor, called implicitly by Python's **Garbage Collector** when an object's reference count drops to 0:

class FileHandler:
    def __init__(self, filename):
        self.file = open(filename, "w")
    def __del__(self):
        self.file.close() # Clean up resources
        print("Resource closed cleanly.")

🎯 Topic Q&A

Q1: What is the difference between `__new__` and `__init__` in Python?

`__new__` is a class-level static method that allocates memory and creates the object instance, returning it. `__init__` is an instance-level initializer that configures attributes on that created instance.

Q2: Why would you override `__new__` instead of `__init__`?

You override `__new__` to control the instantiation process, such as implementing the **Singleton Pattern** (returning an existing instance instead of a new one) or customizing immutable type subclasses (like int or str).

Q3: How does Python handle constructor overloading?

Python does not support traditional constructor overloading (multiple `__init__` methods). If multiple exist, only the latest is registered. We achieve constructor flexibility using default arguments or variable parameters (`*args`, `**kwargs`).

Q4: When does the `__del__` destructor get called in Python?

`__del__` is executed when an object's reference counter hits 0. Because Python uses automatic reference counting and a cycle-detecting garbage collector, the timing of `__del__` is not guaranteed immediately.

Q5: Write a Python class utilizing a classmethod as a custom alternative constructor. 
class Date:
    def __init__(self, day, month, year):
        self.day, self.month, self.year = day, month, year
    @classmethod
    def from_string(cls, date_str):
        d, m, y = map(int, date_str.split('-'))
        return cls(d, m, y) # Dynamic constructor
