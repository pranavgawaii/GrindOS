1. What is Dynamic Programming?

Dynamic Programming (DP) is an optimization technique for problems that have:

  1. Overlapping Subproblems — the same subproblem is solved multiple times
  2. Optimal Substructure — the optimal solution is built from optimal solutions to subproblems

DP avoids redundant computation by storing results (memoization or tabulation).

2. DP vs Plain Recursion

Fibonacci — Naive Recursion (Exponential)

def fib_naive(n):
    if n <= 1: return n
    return fib_naive(n-1) + fib_naive(n-2)
# Time: O(2^n) — computes fib(3) multiple times!

Fibonacci — With Memoization (Linear)

from functools import lru_cache

@lru_cache(maxsize=None)
def fib_memo(n):
    if n <= 1: return n
    return fib_memo(n-1) + fib_memo(n-2)
# Time: O(n), Space: O(n) for call stack + cache

Fibonacci — Bottom-Up DP (Linear, O(1) Space)

def fib_dp(n):
    if n <= 1: return n
    prev, curr = 0, 1
    for _ in range(2, n+1):
        prev, curr = curr, prev + curr
    return curr
# Time: O(n), Space: O(1)

3. The 5-Step DP Framework

  1. Define State: What does dp[i] represent?
  2. Base Case: What are the trivial answers?
  3. Transition: How does dp[i] relate to dp[i-1], dp[i-2], etc.?
  4. Order of Computation: Bottom-up or top-down?
  5. Final Answer: Which cell of the DP table is the answer?
